================================================================================
  SECTION 1 — GAME OVERVIEW
================================================================================

  Genre:        2D top-down survival (Vampire Survivors-style)
  Platform:     PC (Steam primary), optional mobile port post-launch
  Win:          Survive 30 minutes without Chaos reaching 100%
  Lose:         Chaos Meter reaches 100% at any point
  Session:      Single run, 30 minutes, no save mid-run
  Progression:  Within a single run only (XP → level up → pick upgrade)

  CORE LOOP:
    Children spawn → children remove books from shelves → books land on floor
    → Chaos rises → librarian collects books → librarian returns books to shelf
    → XP awarded → Chaos reduced → player levels up → picks upgrade
    → repeat with more children and more chaos until Closing Time (30:00)

  CORE TENSION:
    Should the player clean existing floor books (safe, steady chaos reduction)
    or chase children before they create new messes (risky, higher XP reward)?
    Both strategies must remain viable throughout the run.

================================================================================
  SECTION 2 — TECH STACK
================================================================================

  ENGINE:         Godot 4.x (recommended)
                  - Built-in 2D physics, navigation, signal system
                  - Free, MIT license, no royalties
                  - GDScript is fast to prototype with
                  ALTERNATIVE: Unity 2D with C# (larger community, more assets)
                  All pseudocode in this document uses Godot/GDScript conventions.

  LANGUAGE:       GDScript (Godot) or C# (Unity)

  NAVIGATION:     Godot NavigationServer2D with NavigationRegion2D
                  - Used for child pathfinding to shelves and flee paths
                  - Bake navmesh on scene load; rebake if map changes

  PHYSICS:        Godot CharacterBody2D for librarian and children
                  Godot Area2D for pickup radius, flee radius, chaos zones

  UI:             Godot CanvasLayer + Control nodes
                  - HUD is a separate CanvasLayer (always on top)
                  - Upgrade picker is a separate CanvasLayer (modal)

  PERSISTENCE:    Godot ConfigFile for settings
                  JSON file for high score / run history

  VERSION CONTROL: Git + GitHub (private repo)
                   Branching: main (stable) + dev (active) + feature/* branches

  PROJECT MGMT:   GitHub Issues — one issue per checkbox task in Section 18

  ART TOOLS:      Aseprite (pixel art sprites and animations)
                  Tiled or Godot TileMap editor (map layout)

  AUDIO TOOLS:    Audacity (SFX editing and export)
                  LMMS or GarageBand (music composition)
                  Export all audio as OGG Vorbis for Godot

  PLAYTESTING:    Google Forms (post-session survey)
                  OBS Studio (session recording)

================================================================================
  SECTION 5 — TECHNICAL ARCHITECTURE
================================================================================

  AUTOLOADS (Singletons — registered in Project Settings → Autoload)
  ─────────────────────────────────────────────────────────────

  ChaosManager.gd
    Purpose: Single source of truth for chaos state
    Properties:
      chaos_percent: float           # 0.0 to 100.0
      floor_book_count: int          # updated by BookManager
      cluster_bonus: float           # calculated from cluster detection
      passive_chaos_rate: float      # base rate per second from elite children
    Methods:
      add_chaos(amount: float)
      reduce_chaos(amount: float)
      get_chaos_rate() -> float      # total chaos per second right now
      update_from_books(count: int, cluster_bonus: float)
    Signals:
      chaos_changed(new_percent: float)
      chaos_critical()               # fires when > 80%
      chaos_maxed()                  # fires at 100% → game over

  GameManager.gd
    Purpose: Run state, timer, win/lose, score
    Properties:
      run_time: float                # seconds elapsed
      score: int
      books_shelved: int
      children_intercepted: int
      peak_chaos: float
      is_running: bool
    Methods:
      start_run()
      end_run(won: bool)
      add_score(amount: int)
      get_run_time_string() -> String  # returns "MM:SS"
    Signals:
      run_started()
      run_ended(won: bool)
      closing_time_triggered()

  EventManager.gd
    Purpose: Schedule and fire special events and boss events
    Properties:
      event_queue: Array             # sorted by trigger_time
      active_event: Dictionary       # currently running event or null
    Methods:
      schedule_events()              # called on run start, loads events.json
      tick(delta: float)             # called every frame, checks trigger times
      trigger_event(event_id: String)
      end_event()
    Signals:
      event_started(event_data: Dictionary)
      event_ended(event_id: String)

  UpgradeManager.gd
    Purpose: Manage available upgrades, active upgrades, apply effects
    Properties:
      upgrade_pool: Array            # all upgrades from upgrades.json
      active_upgrades: Dictionary    # upgrade_id -> stack_count
    Methods:
      get_random_upgrades(count: int) -> Array   # returns 3 upgrade dicts
      apply_upgrade(upgrade_id: String)
      get_effect_value(upgrade_id: String) -> float
      has_upgrade(upgrade_id: String) -> bool
    Signals:
      upgrade_applied(upgrade_id: String)

  AudioManager.gd
    Purpose: Centralized audio playback with state-driven music
    Methods:
      play_sfx(sfx_name: String)
      play_music(track_name: String, crossfade: bool)
      set_chaos_ambience_level(chaos_percent: float)
    NOTE: Music layers in/out dynamically based on chaos level.

  ─────────────────────────────────────────────────────────────
  SIGNAL FLOW DIAGRAM
  ─────────────────────────────────────────────────────────────

  Book shelved:
    Bookshelf → emits book_shelved(genre)
    → Librarian.gd receives → removes from inventory → awards XP
    → ChaosManager.reduce_chaos(CHAOS_REDUCTION_PER_BOOK)
    → GameManager.add_score(SCORE_PER_BOOK)
    → BookManager notified → decrements floor_book_count

  Child intercepted:
    Librarian.gd detects child in body_entered
    → Child.drop_book() → Book spawned on floor
    → Librarian auto-picks up (PickupRadius fires)
    → XP bonus awarded (INTERCEPT_XP_BONUS)
    → AudioManager.play_sfx("intercept")

  Chaos changes:
    ChaosManager.chaos_changed(new_percent)
    → HUD.gd updates chaos bar fill and color
    → AudioManager.set_chaos_ambience_level(new_percent)
    → if new_percent >= 100: GameManager.end_run(false)

  Level up:
    Librarian.gd XP threshold crossed
    → GameManager pauses (get_tree().paused = true)
    → UpgradePicker shown with 3 random upgrades
    → Player selects → UpgradeManager.apply_upgrade(id)
    → get_tree().paused = false

  ─────────────────────────────────────────────────────────────
  KEY DESIGN PATTERNS
  ─────────────────────────────────────────────────────────────

  Object Pooling (CRITICAL — implement in Phase 1, not Phase 5):
    Book objects must use an object pool. At minute 30, up to 5,000 books
    are active. Instantiating/freeing nodes at that scale causes hitches.
    Implement BookPool singleton or integrate into BookManager.

  Child Type Extension:
    All child types extend BaseChild.gd and override these methods:
      _get_stats() -> Dictionary         # returns this type's base stats
      _on_reach_shelf(shelf: Bookshelf)  # behavior at shelf
      _get_flee_direction() -> Vector2   # default: away from librarian
      _on_book_dropped()                 # called when intercept occurs

  Data-Driven Tuning:
    ALL numeric values (speeds, radii, chaos rates, XP values) must come
    from constants defined in a single GameConstants.gd or from JSON files.
    NEVER hardcode magic numbers in logic scripts. This enables tuning
    without code changes during balance pass.

  ─────────────────────────────────────────────────────────────
  BOOK SCHEMA
  ─────────────────────────────────────────────────────────────
  {
    "id": String,                    # unique e.g. "book_science_001"
    "genre": String,                 # must match a Bookshelf.genre value
    "state": Enum,                   # SHELVED | ON_FLOOR | CARRIED
    "chaos_weight": float,           # default 1.0; rare books may be 0.5
    "is_rare": bool,                 # rare editions (Summer Reading event)
    "carried_by": NodePath | null    # reference to child carrying it
  }

  Valid genres: "science", "fiction", "history", "children",
                "reference", "biography", "mystery", "rare"

  ─────────────────────────────────────────────────────────────
  CHILD SCHEMA (child_types.json)
  ─────────────────────────────────────────────────────────────
  {
    "type_id": String,               # "curious_child", "speed_reader", etc.
    "display_name": String,
    "base_move_speed": float,        # pixels per second
    "flee_speed_multiplier": float,  # applied to base_move_speed when fleeing
    "flee_radius": float,            # pixels — librarian detection range
    "boldness": float,               # 0.0 (fully flees) to 1.0 (ignores librarian)
    "carry_probability": float,      # 0.0–1.0 chance to carry vs local drop
    "carry_distance_max": float,     # max pixels from shelf to drop destination
    "books_per_visit": int,          # how many books removed per shelf visit
    "chaos_passive_rate": float,     # chaos/sec contributed passively (elite only)
    "spawn_weight_early": float,     # relative spawn weight minutes 0–10
    "spawn_weight_mid": float,       # relative spawn weight minutes 10–20
    "spawn_weight_late": float,      # relative spawn weight minutes 20–30
    "xp_intercept_bonus": int        # bonus XP for intercepting this type
  }

    "spawn_weight_early": 0.2,
    "spawn_weight_mid": 0.5,
    "spawn_weight_late": 0.8,

================================================================================
  SECTION 9 — CHILD AI BEHAVIOUR (PSEUDOCODE)
================================================================================

  All children use a simple state machine. States are:
    IDLE, MOVING_TO_SHELF, AT_SHELF, CARRYING_BOOK, DROPPING_BOOK, FLEEING

  ─────────────────────────────────────────────────────────────
  BASE CHILD LOGIC (all types start here, then override)
  ─────────────────────────────────────────────────────────────

  func _physics_process(delta):
    check_flee_condition()           # overrides current state if true
    match current_state:
      IDLE:       handle_idle()
      MOVING_TO_SHELF: handle_moving_to_shelf()
      AT_SHELF:   handle_at_shelf()
      CARRYING_BOOK: handle_carrying()
      DROPPING_BOOK: handle_dropping()
      FLEEING:    handle_fleeing()

  func check_flee_condition():
    if librarian is within flee_radius AND boldness_check_passes():
      current_state = FLEEING
      flee_direction = position - librarian.position  # away from librarian

  func boldness_check_passes() -> bool:
    # boldness 0.0 = always flee, 1.0 = never flee
    effective_boldness = boldness * (1.0 - current_global_boldness_modifier)
    return randf() > effective_boldness

  func handle_idle():
    idle_timer -= delta
    if idle_timer <= 0:
      target_shelf = find_nearest_active_shelf()
      if target_shelf:
        navigate_to(target_shelf.position)
        current_state = MOVING_TO_SHELF

  func handle_moving_to_shelf():
    move_along_nav_path()
    if arrived_at_target():
      current_state = AT_SHELF

  func handle_at_shelf():
    # override per child type
    remove_book_from_shelf()
    if randf() < carry_probability:
      current_state = CARRYING_BOOK
      drop_destination = choose_carry_destination()
    else:
      drop_book_locally()           # drops adjacent to shelf
      current_state = IDLE
      idle_timer = randf_range(1.0, 3.0)

  func handle_carrying():
    navigate_to(drop_destination)
    if arrived_at_target():
      current_state = DROPPING_BOOK

  func handle_dropping():
    carried_book.state = Book.State.ON_FLOOR
    carried_book.drop_at(position)
    carried_book = null
    current_state = IDLE

  func handle_fleeing():
    flee_target = position + (flee_direction.normalized() * 200.0)
    navigate_to(flee_target)
    if librarian outside flee_radius:
      current_state = IDLE          # stop fleeing when safe

  func on_librarian_touch():        # called by Librarian when bodies overlap
    if carried_book:
      drop_book_at(position)        # book lands here
      emit_signal("intercepted", self)
    # child continues in FLEEING state

  ─────────────────────────────────────────────────────────────
  CURIOUS CHILD overrides:
    carry_probability: 0.5
    books_per_visit: 1
    No overrides to base logic — this IS the base behavior.

  SPEED READER overrides:
    base_move_speed: 120.0
    carry_probability: 0.85
    choose_carry_destination():
      # prefers far zones: reading tables, study areas, opposite end of map
      return farthest_available_zone()

  BOOK FORT BUILDER overrides:
    handle_at_shelf():
      for i in books_per_visit:     # removes 2–4 books
        remove_book_from_shelf()
        drop_book_at(position + Vector2(randf_range(-30, 30), randf_range(-30, 30)))
      current_state = IDLE
    # never carries — always drops at shelf creating local pile

  HIDE AND SEEK KID overrides:
    func navigate_to(target):
      # adds random waypoints to path for zigzag movement
      waypoints = [random_point_1, random_point_2, target]
      navigate_via_waypoints(waypoints)
    choose_carry_destination():
      return random_walkable_point_on_map()   # scatter pattern

  SUGAR RUSH KID overrides:
    base_move_speed: 180.0
    books_per_visit: 1
    handle_at_shelf():
      remove_book_from_shelf()
      drop_book_locally()
      current_state = IDLE          # immediately leaves for next shelf
      idle_timer = 0.2              # very short idle

  TEEN INFLUENCER overrides:
    On spawn: create ChaosZone Area2D around self (radius 150px)
    ChaosZone effect: children within zone get +0.3 chaos_passive_rate
    On intercept/removal: destroy ChaosZone
    Movement: standard base behavior

  ─────────────────────────────────────────────────────────────
  HERDING MECHANIC DETAIL
  ─────────────────────────────────────────────────────────────

  The librarian's movement passively influences child flee vectors.
  There is no explicit "herd" button — it emerges from movement.

  When child is in FLEEING state:
    flee_direction = normalize(child.position - librarian.position)
    flee_target = child.position + flee_direction * 200.0
    navigate_to(flee_target)

  Player can exploit this by:
    - Approaching from one side to push child away
    - Positioning to funnel flee paths toward corners
    - Circling around a group to corral them

  Children in FLEEING state CANNOT remove books from shelves.
  This is the core value of herding — temporary disruption prevention.

================================================================================
  SECTION 10 — CHAOS SYSTEM LOGIC
================================================================================

  CHAOS CALCULATION (runs every frame in ChaosManager._process):

  func _process(delta):
    # Base chaos from floor books
    base_rate = floor_book_count * CHAOS_RATE_PER_BOOK

    # Cluster bonus
    cluster_count = detect_clusters()
    cluster_rate = cluster_count * base_rate * (CHAOS_CLUSTER_MULTIPLIER - 1.0)

    # Passive chaos from elite children
    total_passive = sum of chaos_passive_rate for each active elite child

    # Event multiplier (set by EventManager during active events)
    event_mult = active_event_chaos_multiplier  # default 1.0

    # Total chaos rate this frame
    total_rate = (base_rate + cluster_rate + total_passive) * event_mult

    add_chaos(total_rate * delta)

  CLUSTER DETECTION:
    func detect_clusters() -> int:
      clusters = 0
      checked = {}
      for each floor_book in all_floor_books:
        if floor_book in checked: continue
        nearby = get_books_within_radius(floor_book.position, CHAOS_CLUSTER_RADIUS)
        if nearby.size() >= CHAOS_CLUSTER_THRESHOLD:
          clusters += 1
          checked.merge(nearby)
      return clusters

  NOTE: Cluster detection is expensive at high book counts.
  Run it every 0.5 seconds (not every frame) and cache the result.

  CHAOS REDUCTION:
    Book shelved:     chaos -= CHAOS_REDUCTION_PER_BOOK
    Child intercepted: chaos -= CHAOS_INTERCEPT_REDUCTION
    Objective complete: chaos -= objective.chaos_reduction_reward
    Certain upgrades: passive chaos reduction rate

  CHAOS VISUAL STATES (update HUD colors):
    0–49%:   Green    — calm
    50–69%:  Yellow   — warning
    70–89%:  Orange   — danger
    90–99%:  Red      — critical (bar pulses)
    100%:    Game Over

================================================================================
  SECTION 19 — RISK LOG
================================================================================

  RISK: Chaos curve balance [HIGH]
    The 500→5,000 book ramp is a design target, not a validated number.
    All chaos values MUST be in GameConstants.gd or JSON — never hardcode.
    Mitigation: expose every rate/ramp value as a data constant from day 1.

  RISK: Herding discoverability [HIGH]
    Players may not intuitively understand the flee radius mechanic.
    Mitigation: First run shows a brief animated hint (skippable).
    "Children are afraid of librarians — use it to your advantage."
    Validate in alpha playtest: can players explain herding after 1 run?

  RISK: Performance at minute 30 [HIGH]
    5,000+ active Book nodes is a large scene object count.
    Mitigation: object pool MUST be built in Phase 1. Retrofitting is painful.
    Target: 60fps with 5,000 books and 80 children on screen.
    Fallback: cull books more than 3 screens from player if needed.

  RISK: Automation upgrade power creep [MEDIUM]
    Return Robot + Assistant Librarian + Volunteer Student stacked may
    trivialize the shelving loop entirely.
    Mitigation: automation cap (max 2 automation upgrades per run).
    Also tune automation unit inventory caps conservatively.

  RISK: Child navigation at scale [MEDIUM]
    80 children all running NavigationAgent2D simultaneously may cause
    navigation server load spikes.
    Mitigation: stagger navigation updates (not all agents update same frame).
    Use NavigationAgent2D.set_navigation_layer_value for performance tuning.

  RISK: Boss event difficulty calibration [LOW]
    Boss events may feel too easy early or unfair late.
    Mitigation: all boss event durations and effects in events.json.
    Tune with playtest data in week 18. Target: each boss ~20% harder than
    the previous one.
